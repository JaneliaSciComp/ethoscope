Ethoscope Software

Overview

The platform relies on custom software packages. The software repository is available on github. A technical documentation. on how to set-up the software is also available.

In this page, we simply summarise how the software operates. A FIGURE HERE!!!

  • The Node runs:
    • A web server, "ethoscope_node", which provides a user interface to detect, start, stop... Ethoscopes, on port 80.
    • A data synchronisation tool, "ethoscope_backup", which saves locally data from all Ethoscopes.
    • A update tool "ethoscope_update_node" which allows to update the software of the Node and of the Ethoscopes, on port 8888.
    • A git server to mirror upstream updates from github.
    • An ntp server to synchronise the clock of all Ethoscopes.
  • The Router:
    • Allocate Dynamic IP addresses to detected Ethoscopes.
    • Allocate a reserved IP for the Node.
  • Ethoscopes run:
    • A webserver, "ethoscope_device", on port 9000. It exposes status, tracking data, snapshot...This server is normally only accessed indirectly through the node server.
    • The Ethoscope server can be queried to start/stop video tracking. Importantly, they analyse raw video after the themselves.
    • A MySQL database where all secondary data (e.g. shape and position of individual animals) resulting from the ongoing experiment are stored.
